Lesson 1: Introduction to Computer Programming

Understand the fundamental concepts of programming — what it is, how programming languages work, and the types of errors you will encounter.

60 minWeek 01Theory

1.1 What is Programming?

Programming is the process of creating a set of instructions that tells a computer what to do. You can think of it like writing a recipe for a computer to follow.

The instructions you write are called code, and the person who writes them is a programmer.

Algorithm

An algorithm is a step-by-step process for solving a problem or finishing a task. It's like a plan or a blueprint.

💡

Example: Algorithm for adding two numbers
1. Input Number 1
2. Input Number 2
3. Calculate: Answer = Number1 + Number2
4. Print Answer

1.2 Programming Languages

Programming languages help us give instructions to a computer in a clear and exact way, so the computer knows exactly what to do without confusion.

Key Components of a Programming Language

1. SYNTAX (Form)

Syntax means the rules and structure of a programming language. It tells you how to write the code correctly.

  • Like grammar in English. If grammar is wrong, the sentence doesn't make sense.
  • Same way, if syntax is wrong, the program won't run.

2. SEMANTICS (Meaning)

Semantics means the meaning and behavior of the code. It tells you what the code actually does when it runs.

  • Even if syntax is correct, the program can still give a wrong result if the meaning is wrong.
i

Example: If you write code to add two numbers:
• Syntax = Did you write it in the correct format?
• Semantics = Does it really add the numbers and give the right answer?

1.3 Computer Language Hierarchy

1. Low-Level Languages

Low-level languages are closer to the hardware and harder for humans to read.

  • a) Machine Language (Binary) — The only language computers directly understand. Written in 0s and 1s.
  • b) Assembly Language — Uses short symbolic codes (mnemonics) instead of binary. Requires an assembler to convert to machine code.

2. High-Level Languages

High-level languages are close to human language and much easier to write and read.

Characteristics:

  • Closer to English
  • Easier to write, read, and maintain
  • Needs a compiler or interpreter to run
  • a) Structured / Procedural Languages — e.g., C, Pascal. Code runs in a sequence of steps.
  • b) Object-Oriented Languages — e.g., Python, Java, C++. Code is organized around objects and classes.

Compiler vs Interpreter

CompilerInterpreter
Translates the entire program at onceTranslates and executes one line at a time
Produces an executable fileNo separate executable file is produced
Faster execution after compilationSlower — translates each time
Examples: C, C++Examples: Python, JavaScript

1.4 Types of Errors

Syntax Errors

Errors in the structure/grammar of the code. Detected by the compiler or interpreter before the program runs.

Python
# Syntax Error Example
if x = 5:     # Wrong: should be ==
    print(x)

Logical Errors

The program runs but produces the wrong result because the logic is incorrect. Cannot be detected by the compiler — must be found by humans.

Python
# Logical Error Example
# ERROR: Wrong formula for average
average = (a + b + c) / 2  # Should be divided by 3!

# CORRECT:
average = (a + b + c) / 3
!

Logical errors are hidden — the program runs without crashing but gives wrong output. Always test your programs with known inputs.

Runtime Errors

Errors that occur while the program is running. The program crashes or behaves unexpectedly.

Common causes: memory overflow, division by zero, invalid input.

Python
# Runtime error: Division by zero
result = 10 / 0  # ZeroDivisionError

1.5 Software Testing

Software testing is the process of evaluating a program to find and fix errors before release.

  • Unit Testing — Testing individual parts of a program
  • Integration Testing — Testing how parts work together
  • System Testing — Testing the complete system
✓

Good programmers always test their code with different inputs — including edge cases — to catch all types of errors.
